추가, 변경한 파일을 스테이징하기

me@work MINGW64 ~

git add [파일명]
git add .

add 뒤 .은 전체 파일을 의미합니다.

파일의 상태 확인하기

me@work MINGW64 ~

git status

커밋 생성하기

me@work MINGW64 ~

git commit -m "커밋메세지"

커밋 메세지는 누가 보더라도 어떤 작업 내용이 담겼는지 알아볼 수 있어야 합니다.

스테이징과 커밋 동시에 실행하기

me@work MINGW64 ~

git commit -am "커밋메세지"

단, 새로 생성한 파일은 사용 불가합니다.

커밋 내역 확인하기

me@work MINGW64 ~

git log


커밋의 해쉬를 볼 수 있습니다.

커밋 메세지 수정하기

me@work MINGW64 ~

git commit -amend -m "수정할 커밋 메세지"

커밋 되돌리기

me@work MINGW64 ~

git reset [옵션] [커밋 해쉬]

옵션에 따라 하는 작업이 달라집니다. (옵션을 생략하면 --mixed 옵션이 적용됩니다.)
1. --soft : 로컬 저장소에서 staging area까지 리셋합니다. 즉,리셋을 하면 파일은 스테이징된 상태입니다.
2. --mixed : 로컬 저장소에서 working directory까지 리셋합니다. staging area도 리셋합니다. 3. --hard : 작업 내용까지 완전히 삭제합니다.
