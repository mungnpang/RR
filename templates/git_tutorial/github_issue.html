Github issue?

발견된 버그, 추가할 기능, 프로젝트 기획 등 프로젝트에 발생하는 다양한 이슈들을 관리합니다.

1. issue 생성하기

New issue 클릭합니다.
Labels 는 각 이슈가 어떤 성격을 가졌는지 나타내는 기능입니다.
Milestones는 유사한 이슈들을 하나로 모아 일정 관리를 할 수 있는 기능입니다.

2. issue 작성하기

오른쪽을 보면 이슈와 관련된 다양한 기능을 확인할 수 있습니다.
Assignees로 해당 이슈의 책임자를 지정합니다.
Labels로 해당 이슈와 관련된 라벨을 붙입니다.
Milestone로 해당 이슈가 속한 그룹을 지정합니다.

3. Milestones 생성하기

Milestone에선 각 작업의 진행 사항을 한 눈에 확인할 수 있습니다.
